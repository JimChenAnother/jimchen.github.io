一步步带你理解Laravel请求到响应的生命周期（五）

发表于 2017-05-06   |   分类于 技术类   |

外观注册

在Laravel中很多时候都会用到外观别名，通过外观别名来调用实例的属性和方法，那么接下来我们将了解这个过程Laravel是怎么处理的。

文件Illuminate\Foundation\Bootstrap\RegisterFacades.php

首先进行初始化工作，然后通过外观自动加载类AliasLoader的实例进行注册，AliasLoader的实例需要通过getInstance()实现，在此期间外观别名通过参数传入，外观别名通过$app->make('config')->get('app.aliases')获得，然后AliasLoader实例通过register()注册外观别名。

注意这里$app->make('config')->get('app.aliases')，其实就是获取我们在配置加载时把config目录下所有配置加载到服务容器上的app.php的配置数组的aliases元素，由于配置加载优先执行，所以此时就能从服务容器中获取。

从这里我们可以看出通过后期静态绑定，实现的单例模式，而且允许动态注册更多的外观别名。

紧接着我们知道调用了register()方法，就会向自动加载栈中的首部添加一个新的加载函数，即是通过spl_autoload_register([$this, 'load'], true, true);实现，由于该外观加载函数时注册到了自动加载栈的首部，所以每次当一个类被自动加载时都会先经过外观加载函数处理，那么该外观加载函数就是load($alias)，它的工作就是为外观类设置一个别名，当用户通过外观别名访问类时，能够解析并返回对应的外观类。

举一个栗子：
在~app\Http\目录下路由文件，我们会经常使用到路由的外观Route。当我们使用Route::get('path', 'operation')操作时，程序首先要调用自动加载函数把Route类加载，此时第一个自动加载函数就是之前说的load($alias)，通过该函数找到了Route是一个外观别名，其真实的类是Illuminate\Support\Facades\Route类，于是加载这个类，然后调用静态方法get()，可是该类中是没有该静态方法的，同时，该类继承于Illuminate\Support\Facades\Facade，进一步寻找发现父类也没有，但是父类实现了魔术方法__callStatic($method, $args)，此时__callStatic($method, $args)获得调用，我们还是看一下源码：

__callStatic($method, $args)调用时，首先执行getFacadeRoot()时，那么在此过程中，首先获得后期静态绑定子类实现的getFacadeAccessor()方法返回的结果，由于Illuminate\Support\Facades\Route类就只实现了getFacadeAccessor()方法并返回’router’，于是通过resolveFacadeInstance($name)从服务容器获得对应的路由实例（该实例就是在两个基础服务提供者注册时绑定到了服务容器上），该实例对应的类就是Illuminate\Routing\Router，那么该类中就有get()方法，就会在__callStatic($method, $args)中被调用并返回。

服务提供者注册

当Laravel程序运行过程中，肯定需要更多的服务，比如数据库服务，session服务等，而之前我们说到，在请求实例化之前只会注册两个基础服务提供者并启动，这是远远不够的，所以这里讲到的，就是后期不断使用到的服务。

文件Illuminate\Foundation\Bootstrap\RegisterProviders.php

文件Illuminate\Foundation\Application.php

在这里首先说明的是，服务提供者分三类：

when：当特定事件发生时才会注册。
eager：直接注册。
deferred：会将服务提供者存储在列表中，需要时再注册。

那么我们接着看，在Laravel中，会通过一个缓存文件维护着三类服务提供者信息，该文件在~bootstrap\cache\services.json，而该文件路径就是通过getCachedServicesPath()获得，接着就会实例化一个提供者仓库来注册服务提供者。简单介绍一下服务提供者构造函数参数，首先传入服务容器的一个实例，接着传入文件操作组件的一个实例，再传入缓存文件路径。

文件Illuminate\Foundation\ProviderRepository.php

那么接下来从上面可见，通过调用load($providers)方法，添加了app.php文件中关于服务提供者的数组。我们看到load($providers)方法中首先是调用loadManifest()方法把缓存文件service.json的json数组解析成PHP数组加载进来，但是该文件只是存放上一次运行时服务提供者加载的情况，所以接下来需要与当前服务提供者进行对比，通过shouldRecompile($manifest, $providers)进行对比，如果有需要就会更新缓存文件的内容，没有需要就直接使用缓存得到的服务提供者列表进行注册。从接下来我们都可以清晰知道，

这是对当特定事件发生时才会注册的服务提供者进行事件绑定。

这是直接注册的服务提供者，通过createProvider($provider)直接实例化提供者，然后通过服务容器进行注册。

这是在列表中记录延迟注册的服务提供者，当需要时再注册。

文件Illuminate\Foundation\Application.php

由此我们可见，通过服务容器来维护一个$deferredServices数组，专门存储延迟注册的服务提供者。

总结：外观注册其实就是为特定的类通过class_alias()方法设置一个别名，而使得外观类可以静态化调用就需要实现魔术方法__callStatic($method, $args)进行实例化调用；而服务提供者注册其实就是实例化服务提供者并调用该实例的register()方法，将服务绑定到服务容器实例中，故此Laravel中最核心的还是服务容器。
